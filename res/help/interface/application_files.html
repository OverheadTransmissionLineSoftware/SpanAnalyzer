Applicaton Files

SpanAnalyzer uses a few different file types to keep everything straight.

Config File

The config file holds the user settings, such as:

  • Window size and layout
  • Unit System
  • Plot colors

The file location depends on the platform and user.

  • Linux: $HOME/.config/otls/span-analyzer/spananalyzer.conf
  • Windows: %AppData%\OTLS\SpanAnalyzer\spananalyzer.conf


If this file is not available on startup, a new one will be created.

Data File

SpanAnalyzer stores data that is loaded on every startup. This includes:

  • Cable File Paths
  • Weathercases
  • Analysis Filters

The data file is stored alongside the config file (above) by default, but that can be changed in the application preferences. If this file is not available on startup, a new one will be created.

Cable Files

Cables are stored in separate files so that adding to and removing from the program is as modular as possible. Cable files have an extension of '.cable'. These files can be stored anywhere on the system.

Only the cable filepath is stored in the application data file. On startup, SpanAnalyzer will load all cable files that it knows about. Use caution when moving cable files that may be referenced in the application data.

Document Files

SpanAnalyzer uses document files so projects can be closed and reopened at a later time. Document files have an extension of '.spananalyzer'. These files can be stored anywhere on the system.

The document file stores details for the spans, including:

  • Name
  • Cable
  • Constraint
  • Stretch Weathercases
  • Span Length / Elevation Change

The document file references application data, such as cables and weathercases. If any of the referenced data is removed, it may cause problems next time the document is loaded. Use caution when removing application data that may be referenced in document files.
